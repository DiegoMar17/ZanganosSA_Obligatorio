## Trabajo realizado

### 1. Fix: ColmenasController.Editar — pérdida de datos en UPDATE

**Problema:** El POST de edición usaba `_ctx.Colmenas.Update(colmena)` con el modelo vinculado directamente del formulario. Como el formulario `Editar.cshtml` no incluía campos ocultos para `AsignadoAId`, `AsignadoNombre` ni `UltimaVisita`, cada edición normal de colmena silenciosamente borraba esos valores.

**Solución aplicada en `ColmenasController.cs`:**
- Se carga la entidad existente con `_ctx.Colmenas.Find(id)`
- Se copian campo a campo solo los valores editables del formulario
- `AsignadoAId`, `AsignadoNombre` y `UltimaVisita` se preservan tal cual

```csharp
var existente = _ctx.Colmenas.Find(id);
existente.Codigo = colmena.Codigo;
existente.ApiarioId = colmena.ApiarioId;
// AsignadoAId / AsignadoNombre / UltimaVisita no viajan en el form: se preservan
_ctx.SaveChanges();
```

### 2. Fix: ColmenasController.Crear — permisos extendidos a EMPLEADO

**Problema:** `Crear` (GET y POST) era `[Authorize(Roles = "ADMIN")]`. Los empleados no podían registrar nuevas colmenas.

**Solución:**
- Se quitó `[Authorize(Roles = "ADMIN")]` de ambas acciones (el global `AuthorizeFilter` sigue requiriendo login)
- Si el empleado no tiene sector asignado → redirect con `TempData["Error"]`
- Si intenta crear en apiario ajeno → error + redirect
- Al crear, `AsignadoAId`/`AsignadoNombre` se auto-setean al empleado que registra
- `CargarApiarios()` refactorizado: muestra solo el propio sector al empleado, todos al admin
- Botón "+ Nueva colmena" en `Index.cshtml` ahora visible para todos (antes solo ADMIN)

---

### 3. Fix: EquipoController.AsignarApiario — bulk-assign de colmenas preexistentes

**Problema reportado por el usuario:** Al asignar un apiario a un empleado, las colmenas ya existentes en ese sector quedaban "Sin asignar" y el empleado no obtenía control sobre ellas.

**Solución en `EquipoController.cs`:**
```csharp
// Colmenas sin dueño en el nuevo sector → pasan al empleado
var colmenasSinDueno = _ctx.Colmenas
    .Where(c => c.ApiarioId == apiarioId.Value && c.AsignadoAId == null)
    .ToList();
foreach (var c in colmenasSinDueno)
{
    c.AsignadoAId = empleado.Id;
    c.AsignadoNombre = empleado.NombreCompleto;
}
```
- Solo afecta colmenas sin responsable (no roba las asignadas a otro compañero de sector)
- La limpieza de colmenas fuera del nuevo sector (lógica previa) se mantiene

### 5. Comparación de bases de datos

Se realizó análisis completo entre:

| | `colmena.db` (SQLite/EF Core) | SQL Server original del equipo |
|---|---|---|
| Tablas | 14 propias + 7 Identity | 17 tablas |
| Normalización | Media (denorm. deliberada) | Alta (FKs estrictas, pivots) |
| Usuarios | ASP.NET Identity completo | No modelado |
| Auditoría | Completa | No modelada |
| Inventario | Solo stock actual | Con historial de movimientos |
| Control sanitario | Por apiario | Por colmena individual |
| Cosecha | Con Vendida/PrecioPorKg inline | Vinculada a colmena + FK a Finanza |
| Traslados | Solo cantidad total, strings | Pivot Traslado_Colmena, FKs reales |

**Tablas exclusivas de SQL Server original:** `Apicultor`, `Propietario_Predio`, `Evaluacion_Colmena`, `Traslado_Colmena`, `Movimiento_Inventario`, `Visita_Tarea`, `Tarea_Colmena`, `Tarea_Apiario`

**Tablas exclusivas de colmena.db:** `Auditorias`, `HistorialesAcceso`, `AlertasComunitarias`, `NotificacionesAlerta` + 7 Identity

### 6. Diseño de base de datos unificada v3 (T-SQL / SQL Server)

**Decisiones de diseño:**

| Decisión | Razonamiento |
|---|---|
| `Apicultor` → `Apiario.ResponsableId` FK a `AspNetUsers` | Admin = apicultor en empresa propia; FK real honra normalización sin tabla redundante |
| `Propietario_Predio` → tabla `Propietarios` separada | Entidad externa con datos propios; vale la normalización |
| `EvaluacionesColmena` restaurada | Recupera estado histórico por inspección (normalización SQL Server) |
| `TrasladosColmena` pivot restaurada | Detalle exacto de qué colmenas viajaron |
| `MovimientosInventario` de SQL Server | Sin historial no se auditan pérdidas/entradas |
| Campos `*Nombre` denormalizados eliminados | Se reemplazan por FK + JOIN |
| Denorm en `Auditorias`/`HistorialesAcceso` mantenida | Preserva historial si se borra un usuario |
| Control sanitario: nivel apiario (de colmena.db) | Se mantiene granularidad de apiario |
| Cosecha: con Vendida/PrecioPorKg (de colmena.db) | Más funcional para el flujo de venta |

**Resultado: 28 tablas totales** (21 propias + 7 Identity), 13 índices, 24 FKs
