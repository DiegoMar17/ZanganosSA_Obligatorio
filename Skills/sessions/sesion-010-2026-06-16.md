## 3. Bug real encontrado: fuga de datos de sector en el layout compartido

### Diagnóstico

El panel de notificaciones (campana) y dos badges del sidebar en `Views/Shared/_Layout.cshtml` quedaron sin actualizar cuando se implementó el scoping por sector en los controllers: consultaban `_db.Colmenas`, `_db.Inspecciones` y `_db.Apiarios` de forma global, sin filtrar por `ApiarioAsignadoId` del usuario logueado.

**Confirmado en vivo:** el empleado de prueba (asignado a "El Eucaliptal", apiario id=3) veía en el panel de notificaciones la alerta "Colmena C-82 en estado crítico — La Rinconada", un apiario fuera de su sector. Esto viola directamente el requisito del plan de roles: *"empleado no puede ver datos de otros sectores/empleados"*.

### Fix aplicado

`Views/Shared/_Layout.cshtml`:

```csharp
int? sectorId = esAdmin ? null : _userMgr.Users.Where(u => u.Id == currentUserId).Select(u => u.ApiarioAsignadoId).FirstOrDefault();
var apiariosCount = esAdmin ? _db.Apiarios.Count() : (sectorId.HasValue ? 1 : 0);
var inspeccionesPendientesQ = esAdmin
    ? _db.Inspecciones.Where(i => i.Estado == "pendiente" || i.Estado == "vencida")
    : _db.Inspecciones.Where(i => (i.Estado == "pendiente" || i.Estado == "vencida") && sectorId.HasValue && i.ApiarioId == sectorId.Value);
var colmenasRojoQ = esAdmin
    ? _db.Colmenas.Where(c => c.EstadoSemaforo == "rojo")
    : _db.Colmenas.Where(c => c.EstadoSemaforo == "rojo" && sectorId.HasValue && c.ApiarioId == sectorId.Value);
```

El badge de Apiarios, el badge de Inspecciones pendientes, el contador de "nuevas" del panel y los dos `@foreach` del panel de notificaciones pasaron a usar estas variables ya filtradas.

### Verificación post-fix

| Vista | Antes | Después (empleado) | Admin (sin cambios) |
|---|---|---|---|
| Badge Apiarios | `10` (global) | `1` (su sector) | `10` |
| Badge Inspecciones pendientes | global | `0` (su sector) | sin cambios |
| Panel de notificaciones | mostraba "La Rinconada" (otro sector) | vacío | sigue mostrando "Colmena C-82 — La Rinconada" |

## Decisiones técnicas

| Decisión | Justificación |
|---|---|
| Scoping por sector también en `_Layout.cshtml`, no solo en controllers | El panel de notificaciones y los badges del sidebar se renderizan en *toda* página vía layout compartido; un filtro solo a nivel de controller no alcanza para estos elementos globales |
| Verificación server-side además de ocultar botones en la vista | Ocultar Editar/Eliminar en la vista no es suficiente — se confirmó con tampering directo de POSTs que el controller también rechaza accesos fuera de sector |
| Empleado de prueba con sector real asignado | El empleado de prueba de sesiones anteriores no tenía sector, lo cual no permitía validar el scoping; se creó uno nuevo asignado a un apiario concreto para esta sesión |
