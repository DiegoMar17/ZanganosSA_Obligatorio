## Contexto del problema

El usuario reportó que las vistas del proyecto se veían comprimidas, sin cards, como "cadenas de string estáticas". Específicamente mencionó Apiarios, Perfil, Transhumancia y Producción.

## Diagnóstico

### Causa raíz — CSS incompleto en el proyecto viejo

El usuario estaba corriendo el **proyecto viejo** (`ColmenaEmpresa/ColmenaEmpresa/ColmenaEmpresaMVC/`) que tenía un CSS desactualizado:

| CSS | Tamaño | Clases faltantes |
|---|---|---|
| Proyecto viejo (antes) | 25 KB | `.apiary-card`, `.apiary-grid`, `.prod-row`, `.prod-bar`, `.prod-fill`, `.sem-dots`, `.sm-dot`, `.tl-item`, `.tl-dot`, `.tl-line`, y muchas más |
| Proyecto real | 33 KB | Completo con todas las clases del sistema de diseño |

Sin esas clases CSS, los divs con `.apiary-card` se renderizaban como bloques sin estilo (texto plano), explicando el aspecto de "cadenas de string estáticas".

### Bug secundario en Apiarios/Index (proyecto viejo)

La vista usaba `var(--@item.EstadoSemaforo)` que generaba `var(--rojo)` o `var(--amarillo)`, variables CSS que no existen. El CSS define `--red`, `--amber`, `--green`. Tampoco mapeaba correctamente el texto del pill ("red"/"amber" en lugar de "Insp. vencida"/"Visitar pronto").

### Bug en Produccion/Index (proyecto viejo)

- `Model.Max(m => m.PesoBruto)` dentro del `foreach` sin guard → división por cero si todos los valores son 0
- Sin estado vacío cuando no hay cosechas
- Sin botones Editar/Eliminar

### Perfil/Index (proyecto viejo)

La vista usaba clases propias (`.profile-layout`, `.profile-card`, `.profile-body`, `.profile-stats`, `.pref-row`, `.danger-zone`, etc.) que no estaban definidas en ningún CSS. Reescrita con clases estándar del design system.

### Cambio visual en Produccion/Index (proyecto real)

En la sesión anterior el cambio `b320652` había convertido la vista de producción (proyecto real) de un diseño de cards con barras horizontales a una tabla densa de 9 columnas. Fue revertido a cards con `.prod-row`.

## Decisiones técnicas

| Decisión | Justificación |

| Copiar CSS completo al proyecto viejo | Sincronizar ambos proyectos con el mismo design system en lugar de mantener dos CSS distintos |

| Restaurar `prod-row` en producción real | El diseño en tabla sacrificaba la legibilidad visual por densidad de datos; las barras horizontales comunican mejor la comparación entre apiarios |
