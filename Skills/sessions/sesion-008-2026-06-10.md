## Trabajo realizado

### Análisis comparativo de ideas

- Se evaluaron ambas ideas en criterios: complejidad técnica, valor académico, viabilidad, integración con el modelo de datos existente, relevancia para el rubro apícola y cobertura de requisitos académicos.

- La Idea 2 encajaba directamente con la entidad `APICULTOR` existente y cubre autenticación, autorización y auditoría — conceptos fundamentales de ingeniería de software.

### Diseño del sistema de roles

- Se definieron dos roles: `ADMIN` y `EMPLEADO`.
- Se diseñaron las modificaciones SQL necesarias sobre la entidad `APICULTOR`.
- Se crearon las tablas `AUDITORIA` e `HISTORIAL_ACCESO`.
- Se especificaron los permisos exactos para cada rol sobre todas las entidades del sistema.

## Decisiones técnicas

| Decisión | Alternativa descartada | Razón |
|----------|------------------------|-------|
| Idea 2: Control de Acceso por Roles | Idea 1: Smart Supply Marketplace | Mayor viabilidad, valor académico y encaje con el modelo existente |
| PIN de 6 dígitos con bcrypt | Contraseña alfanumérica | Más simple para el contexto apícola, más fácil de operar en campo |
| Gestor de PINs como panel separado | CRUD integrado al ABM de empleados | Mejor separación de responsabilidades y seguridad |
| Tabla `AUDITORIA` independiente | Campo `modificado_por` en cada tabla | Escalable, centralizado, más limpio para reportes |
