## Decisiones técnicas tomadas

| Decisión | Justificación |
|---|---|
| **C# como único lenguaje de backend** | Establecido como regla permanente del proyecto |
| **EF Core 9.0.5 + SQLite** | Estabilidad, sin servidor externo necesario para desarrollo |
| **`EnsureCreated()` en lugar de migrations** | Simplicidad para esta etapa; sin necesidad de CLI en cada cambio |
| **`DbInitializer.Seed()`** | Mantiene los datos de ejemplo sin hardcodear en controllers |
| **Target framework net9.0** | Downgrade desde net10.0 para compatibilidad estable con EF Core 9.x |
| **`[NotMapped]` en propiedades calculadas** | `PesoNeto`, `PorcentajeStock`, `EstadoStock` no deben persistirse en DB |
| **Inyección de dependencias en controllers** | Patrón correcto MVC; elimina listas estáticas acopladas |
| **Python scripts ignorados** | Los `.py` en `.agents/skills/` son scripts internos de Claude Code, no del proyecto |
| **JavaScript de browser no migrado** | `site.js` es código de interfaz (canvas, DOM, calendario visual) — se ejecuta en el navegador, no puede ser C# |

## Bugs corregidos durante la migración

| Bug | Descripción | Fix aplicado |
|---|---|---|
| Mapeo hardcodeado ApiarioId→Nombre | `switch` con IDs fijos; Paso Carrasco devolvía "Desconocido" en Inspecciones y Sanidad | Reemplazado por `_db.Apiarios.FindAsync(id)?.Nombre` |
| Dashboard desconectado | KPIs hardcodeados (148 colmenas, 12 apiarios) sin reflejar datos reales | Calculados desde DbSets reales |
| Semáforo Detalle Apiario | Comparaba `"red"/"amber"` (inglés) contra valores `"rojo"/"amarillo"` (español) | Corregida la comparación en la vista |
| Inspecciones sobrescribían datos | `TotalColmenas=20`, `ColmenasInspeccionadas=18` hardcodeados en POST | Eliminado; se usa el valor del formulario y el apiario |
| `UltimaVisita` nunca se actualizaba | Registrar una inspección no actualizaba las colmenas | POST de Inspección actualiza `UltimaVisita` de todas las colmenas del apiario |
| `CrearRegistroIngreso` ignorado | Checkbox en cosechas no creaba el RegistroFinanciero | Implementado en `ProduccionController.Crear()` |
| `new Random()` en Transhumancia | Distancia aleatoria si no se ingresaba valor | Eliminado; la distancia es 0 si no se especifica |
| Produccion — apiarios incompletos | El formulario solo mostraba 2 de 4 apiarios | Select dinámico con todos los apiarios de la DB |
| Inventario sin CRUD | Botón "+ Movimiento" no vinculado | Agregadas acciones `Crear` y `AjustarStock` |
