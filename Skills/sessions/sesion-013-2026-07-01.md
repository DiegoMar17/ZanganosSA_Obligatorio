- `options.UseSqlite(...)` → `options.UseSqlServer(connectionString)`

    new Colmena { Codigo="C-01", ApiarioId=monteOlivo.Id, ... },

### 3. Modelo Apiario.cs — campo [NotMapped]

`ApiariosController` usaba `a.TotalColmenas = cols.Count` (calculado en memoria). El campo fue eliminado de la DB pero el controller lo seguía usando, causando error de build.

**Solución:**
```csharp
[NotMapped]
public int TotalColmenas { get; set; }
```
Campo calculado por el controller, nunca mapeado a columna SQL.

## Decisiones técnicas

| Decisión | Razonamiento |
|---|---|
| `[NotMapped]` en `TotalColmenas` | El campo no existe en DB pero `ApiariosController` lo calcula en memoria; evita breaking change en el controller |
| `DeleteBehavior.NoAction` / `SetNull` | Previene ciclos de cascade en SQL Server que no existen en SQLite |
| Seed con `SaveChanges()` intermedio | SQL Server requiere IDs reales antes de usarlos como FK; SQLite los infería |
| `EnsureCreated()` sobre migraciones | Proyecto académico sin historial de migraciones; drop & recreate es el flujo |
| No commitear `bin/obj/colmena.db` | Convención establecida en sesiones anteriores |
