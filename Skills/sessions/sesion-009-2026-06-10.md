## 1. Sistema de Alerta Comunitaria

### Descripción

Módulo permanente que permite reportar amenazas sanitarias o ambientales y notificar automáticamente a todos los apiarios registrados dentro de un radio configurable (km). Usa la fórmula de Haversine para calcular distancias geográficas reales entre coordenadas.

### Artefactos creados

| Archivo | Descripción |
|---|---|
| `Models/AlertaComunitaria.cs` | Entidad principal: tipo de amenaza, lat/lng, radio, estado (activa/resuelta), notificaciones |
| `Models/NotificacionAlerta.cs` | Notificación por apiario: distancia calculada, estado de lectura |
| `Controllers/AlertasController.cs` | CRUD completo + cálculo de proximidad con Haversine en `Crear POST` |
| `Views/Alertas/Index.cshtml` | 4 stat-cards, mapa Leaflet con círculos de radio, lista de alertas con pills de tipo/estado |
| `Views/Alertas/Crear.cshtml` | Layout `.g2`: mapa clickable a izquierda, formulario a derecha; selector de apiario pre-rellena coordenadas |
| `Views/Alertas/Detalle.cshtml` | Info de alerta, mapa de impacto con radio + marcadores, tabla de apiarios notificados con barras de distancia |
| `Data/AppDbContext.cs` | Agregados `DbSet<AlertaComunitaria>` y `DbSet<NotificacionAlerta>` |
| `Program.cs` | `CREATE TABLE IF NOT EXISTS` para las dos tablas nuevas + seed de 2 alertas de ejemplo con notificaciones calculadas |
| `Views/Shared/_Layout.cshtml` | Nuevo ítem Alertas en sidebar con badge rojo dinámico (`Count(a => a.Estado == "activa")`) |

### Fórmula Haversine (C#)

```csharp
private static double Haversine(double lat1, double lon1, double lat2, double lon2)
{
    const double R = 6371;
    var dLat = (lat2 - lat1) * Math.PI / 180;
    var dLon = (lon2 - lon1) * Math.PI / 180;
    var a = Math.Sin(dLat / 2) * Math.Sin(dLat / 2)
          + Math.Cos(lat1 * Math.PI / 180) * Math.Cos(lat2 * Math.PI / 180)
          * Math.Sin(dLon / 2) * Math.Sin(dLon / 2);
    return R * 2 * Math.Atan2(Math.Sqrt(a), Math.Sqrt(1 - a));
}
```

## 2. Persistencia de tema (dark/light) entre páginas

### Diagnóstico

El tema se reseteaba al cambiar de página porque:
- **Proyecto viejo**: el click handler solo cambiaba el DOM en memoria; nunca llamaba `localStorage.setItem()`
- **Proyecto real**: el script de `localStorage.getItem()` estaba _después_ del `<link rel="stylesheet">`, causando un flash del tema incorrecto antes de aplicar el guardado

### Artefactos modificados

| Archivo | Cambio |
|---|---|
| `ColmenaEmpresa/.../wwwroot/js/site.js` | IIFE que lee `localStorage` al cargar + `addEventListener('click')` que guarda con `localStorage.setItem('theme', next)` |
| `ColmenaEmpresa/.../Views/Shared/_Layout.cshtml` | Script inline de `localStorage.getItem()` movido **antes** del `<link rel="stylesheet">` |
| `ColmenaEmpresaMVC/Views/Shared/_Layout.cshtml` | Script inline ya correctamente posicionado antes del CSS; confirmado sin flash |

## 3. Reparación de mapas Leaflet

### Diagnóstico de los dos bugs encontrados

#### Bug 1 — CDN `unpkg.com` poco confiable

Todos los views con mapas usaban `unpkg.com` como CDN para Leaflet. `unpkg.com` es un CDN de npm sin SLA, propenso a caídas. Causa: "mapas no cargan".

#### Bug 2 — Separador decimal roto en locale español (`es-UY`)

En sistemas con locale uruguayo/español, C# renderiza `@Model.Latitud` como `"-34,337"` (coma como separador decimal). Esto producía:

- **`Alertas/Detalle.cshtml`**: el objeto JS `{lat: -34,337}` era inválido — la coma es el operador coma de JavaScript, rompiendo completamente el bloque.
- **`Alertas/Crear.cshtml`**: el `value` del `<option>` era `"-34,337,-56,713"`. Al hacer `split(',')` se obtenían 4 tokens en lugar de 2, asignando `lat=-34` y `lng=337`. Causa: "apiarios en ubicaciones aleatorias".

### Artefactos modificados

| Archivo | Cambio |
|---|---|
| `Views/Apiarios/Index.cshtml` | CDN → jsDelivr |
| `Views/Apiarios/Crear.cshtml` | CDN → jsDelivr; `type="number"` en inputs Latitud/Longitud |
| `Views/Apiarios/Editar.cshtml` | CDN → jsDelivr; `type="number"` en inputs; parsing inicial corregido (`isNaN()` en lugar de `\|\|`) |
| `Views/Alertas/Index.cshtml` | CDN → jsDelivr |
| `Views/Alertas/Crear.cshtml` | CDN → jsDelivr; separator de option value cambiado de `,` a `\|`; JS actualizado a `split('\|')` |
| `Views/Alertas/Detalle.cshtml` | CDN → jsDelivr; `alertaInfo` ahora usa `JsonSerializer.Serialize(new { lat, lng, radio, tipo, titulo })` — garantiza InvariantCulture |

**CDN anterior:** `https://unpkg.com/leaflet@1.9.4/dist/`  
**CDN nuevo:** `https://cdn.jsdelivr.net/npm/leaflet@1.9.4/dist/` (jsDelivr: red global, SRI, 99.9% SLA)

### Fix adicional — inputs de coordenadas

Cambiar a `type="number"` en los inputs de Latitud/Longitud hace que:
- El browser siempre use punto como separador decimal en `.value` (independiente del locale)
- ASP.NET Core use `InvariantCulture` para el model binding de esos campos

## Decisiones técnicas

| Decisión | Justificación |
|---|---|
| `CREATE TABLE IF NOT EXISTS` en lugar de migración EF | La BD ya existe con datos reales; `EnsureCreated()` no agrega tablas nuevas; el bloque try/catch permite re-ejecutar sin errores |
| jsDelivr sobre unpkg | jsDelivr tiene redundancia, SRI integrado y SLA garantizado; unpkg no tiene SLA |
| `JsonSerializer.Serialize()` para embedding de coordenadas en JS | Garantiza InvariantCulture independientemente del culture del sistema operativo del servidor |
| Separador `\|` para coordenadas en option value | Evita colisión con la coma decimal en locales europeos/latinoamericanos |
| Regla permanente: C# único lenguaje backend | Establecida en sesión 004, innegociable |
